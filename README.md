# **NLP Project**

University project comparing how different prompting techniques affect reasoning quality across multiple LLMs.

## Prompting Techniques

- **Zero-shot** — direct question, no examples
- **Few-shot** — 3 demonstration examples before the question
- **Few-shot Chain-of-Thought** — 3 examples with step-by-step reasoning (Wei et al. 2023)
- **Zero-shot Chain-of-Thought** — "Let's think step by step" trigger (Kojima et al. 2023)

## Dataset

[facebook/natural_reasoning](https://huggingface.co/datasets/facebook/natural_reasoning) — 1.1M multi-step reasoning QA pairs with reference answers generated by Llama-3.3-70B.

We use a stratified sample of **100 questions** (`Data/sampled.jsonl`, seed=42):

| Answer Type | Count |
|---|---|
| No answer | 18 |
| Single word | 10 |
| Short (2-20 words) | 37 |
| Long (20+ words) | 35 |

## Models

All models run on Google Colab (T4 GPU) with 4-bit quantization (bitsandbytes NF4).

| Model | Parameters | Notebook | HuggingFace |
|---|---|---|---|
| Llama 3.1 8B Instruct | 8B | `notebook_llama.ipynb` | [meta-llama/Llama-3.1-8B-Instruct](https://huggingface.co/meta-llama/Llama-3.1-8B-Instruct) |
| Mistral 7B Instruct v0.3 | 7B | `notebook_mistral.ipynb` | [mistralai/Mistral-7B-Instruct-v0.3](https://huggingface.co/mistralai/Mistral-7B-Instruct-v0.3) |
| Gemma 4 E4B IT | Е4B | `notebook_gemma.ipynb` | [google/gemma-4-E4B-it](https://huggingface.co/google/gemma-4-E4B) |

## Evaluation

- **Exact Match** — normalized string equality
- **Contains Match** — reference answer appears within model output
- **ROUGE-L** — longest common subsequence overlap
